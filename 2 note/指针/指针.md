# 1 TSharedPtr，TWeakPtr，TUniquePtr

- **`TSharedPtr`**：类似于标准库的`std::shared_ptr`，用于共享所有权的智能指针。
- **`TWeakPtr`**：类似于标准库的`std::weak_ptr`，用于解决循环引用问题。
- **`TUniquePtr`**：类似于标准库的`std::unique_ptr`，用于独占所有权的智能指针。
T看上去就是一个可以模板的操作
# 2 TWeakObjectPtr，TSoftObjectPtr，FScopedPointer

- **`TWeakObjectPtr`**：用于弱引用`UObject`，不会影响Object的GC，避免悬挂指针问题。
- **`FScopedPointer`**：用于在作用域结束时自动销毁对象。
## TWeakObjectPtr实现
文章解释：
https://segmentfault.com/a/1190000044931137

```cpp
template<class T, class TWeakObjectPtrBase>
struct TWeakObjectPtr : private TWeakObjectPtrBase
{
	
}

```
1 首先就是一个私有继承，TWeakObjectPtr中全是方法，没有任何成员变量。TWeakObjectPtrBase这是一个模板类型，一般是默认的FWeakObjectPtr这个类。
2 私有继承也有些好处：
1. Private继承表面是继承，其实本质是一种更加简便达到组合的方式，子类和父类并不是Is-a的关系。完全避免类型兼容原则（子类可被认为是父类），可显示阻止不合理的类型转化（编译期暴露问题）。
2. 使用继承便可以利用上空基类优化（EBO）以减少内存占用。
3. 不希望派生类的子类直接使用基类的任何变量和方法（可以用using改变可见性）。
4. 派生类可以覆盖基类的虚函数，但是不将这些函数暴露给外部。
```cpp
struct FWeakObjectPtr
{
	int32		ObjectIndex = UE::Core::Private::InvalidWeakObjectIndex;
	int32		ObjectSerialNumber = 0;
}
// FWeakObjectPtr的赋值方法
void FWeakObjectPtr::operator=(const class UObject *Object)
{
	if (Object)
	{
		// 拿到ObjectIndex
		ObjectIndex = GUObjectArray.ObjectToIndex((UObjectBase*)Object);
		// 创建ObjectSerialNumber，是一个int32递增的计数器
		ObjectSerialNumber = GUObjectArray.AllocateSerialNumber(ObjectIndex);
	}
	else
	{
		Reset();
	}
}
```
3 FWeakObjectPtr类中只有两个int类型的成员变量，其中ObjectIndex就是GUObjectArray全局数组中的索引。通过ObjectIndex能找到全局数组中的FUObjectItem，进而在FUObjectItem中找到对应的UObject对象。有个ObjectIndex就能找到UObjectItem了，还需要ObjectSerialNumber原因是不同时间下ObjectIndex对应的ObjectItem可能是不同的，这里有分析[[GC#从GUObjectArray数组中移除掉]]。

# 3 TSoftObjectPtr，TStrongObjectPtr

## TSoftObjectPtr
- **TSoftObjectPtr** 是一种重要的资源管理工具，它允许开发者以“软引用”的方式引用资源（如纹理、模型、蓝图等），而不会强制资源立即加载到内存中
- **延迟加载**：仅保存资源路径，不主动加载资源。
- **安全性**：自动处理资源卸载后的引用失效问题。
- **兼容性**：支持蓝图和C++，可与**AsyncLoad**配合实现异步加载。
- **内存优化**：避免不必要的内存占用，按需加载。
字面上的意思，资源可以通过软引用来实现按需加载，在需要用到的时候在加载而不是伴随关卡，或者伴随蓝图中一起加载到内存中。
## TStrongObjectPtr
强引用，不会被gc回收掉，内部实现是通过继承FGCObject，重写ARO方法实现的。
