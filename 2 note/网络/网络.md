# 1 客户端 服务器建立链接
## 1 服务器链接
### 1 Listen
服务器的监听起始点是在LoadMap方法中
```cpp
// Listen for clients.
if (Pending == NULL && (!GIsClient || URL.HasOption(TEXT("Listen"))) && !URL.HasOption(TEXT("DeferredListen")))
{
	if (!WorldContext.World()->Listen(URL))
	{
		UE_LOG(LogNet, Error, TEXT("LoadMap: failed to Listen(%s)"), *URL.ToString());
	}
}
```
如果不是客户端，或者是URL中有Listen，就认为是服务器走到了LoadMap方法，初始化监听。
```cpp
bool UWorld::Listen( FURL& InURL )
{
#if WITH_SERVER_CODE

	// Create net driver.
	// 创建netDriver
	if (GEngine->CreateNamedNetDriver(this, NAME_GameNetDriver, NAME_GameNetDriver))
	{
		NetDriver = GEngine->FindNamedNetDriver(this, NAME_GameNetDriver);
		NetDriver->SetWorld(this);
		FLevelCollection* const SourceCollection = FindCollectionByType(ELevelCollectionType::DynamicSourceLevels);
		if (SourceCollection)
		{
			SourceCollection->SetNetDriver(NetDriver);
		}
		FLevelCollection* const StaticCollection = FindCollectionByType(ELevelCollectionType::StaticLevels);
		if (StaticCollection)
		{
			StaticCollection->SetNetDriver(NetDriver);
		}
	}
	// NetDriver调用InitListen方法
	if( !NetDriver->InitListen( this, InURL, false, Error ) )
	{
		
	}
	return true;
#else
	return false;
#endif // WITH_SERVER_CODE
}
```
整个方法都由WITH_SERVER_CODE宏包装起来，是在UBT中设置的宏。第一部分就是创建NetDriver，创建这里会通过BaseEngine.ini里面的NetDriverDefinitions配置创建出IpNetDriver类，创建完后还有一些和当前world绑定的操作。第二部分就是调用InitListen方法初始化监听
### 2 UIpNetDriver::InitListen
```cpp
// 这个方法就两部分，一个是InitBase方法初始化，一个是InitConnectionlessHandler
bool UIpNetDriver::InitListen( FNetworkNotify* InNotify, FURL& LocalURL, bool bReuseAddressAndPort, FString& Error )
{
	if( !InitBase( false, InNotify, LocalURL, bReuseAddressAndPort, Error ) )
	{
		return false;
	}
	InitConnectionlessHandler();
	LocalURL.Port = LocalAddr->GetPort();
	return true;
}
```
在UNetDriver::InitBase里面(也就是基类里面)，创建了NetConnectionClass（这是个UCLass，不是UObject），是通过BaseEngine.ini里面的NetConnectionClassName配置创建出IpConnection类(这是个UPlayer的子类)的。然后就是设置了FNetworkNotify类型的成员变量Notify，在服务器这部分这个Notify是当前的World，也就是调用Listen的World。最后如果NetConnectionClass创建成功就返回true。
在UIpNetDriver::InitBase里面，看上去就是创建Socket，然后做一些Socket的初始化，
### 3 UNetDriver::InitConnectionlessHandler
```cpp
void UNetDriver::InitConnectionlessHandler()
{
	check(!ConnectionlessHandler.IsValid());
	{
		ConnectionlessHandler = MakeUnique<PacketHandler>(&DDoS);

		if (ConnectionlessHandler.IsValid())
		{
			ConnectionlessHandler->NotifyAnalyticsProvider(AnalyticsProvider, AnalyticsAggregator);
			ConnectionlessHandler->Initialize(Handler::Mode::Server, MAX_PACKET_SIZE, true, nullptr, nullptr, NetDriverName);

			// Add handling for the stateless connect handshake, for connectionless packets, as the outermost layer
			TSharedPtr<HandlerComponent> NewComponent =
				ConnectionlessHandler->AddHandler(TEXT("Engine.EngineHandlerComponentFactory(StatelessConnectHandlerComponent)"), true);

			StatelessConnectComponent = StaticCastSharedPtr<StatelessConnectHandlerComponent>(NewComponent);

			if (StatelessConnectComponent.IsValid())
			{
				StatelessConnectComponent.Pin()->SetDriver(this);
			}

			ConnectionlessHandler->InitializeComponents();
		}
	}
}
```
这个方法里就是创建ConnectionlessHandler和StatelessConnectComponent
## 2 客户端链接

如果是客户端走的Browse方法，里面会创建PendingNetGame，然后会调用InitNetDriver方法来进行与服务器链接。
### 1 UPendingNetGame::InitNetDriver

```cpp
void UPendingNetGame::InitNetDriver()
{
	LLM_SCOPE(ELLMTag::Networking);

	if (!GDisallowNetworkTravel)
	{
		// 开始这和服务器创建一样，都是创建出NetDriver
		if (GEngine->CreateNamedNetDriver(this, NAME_PendingNetDriver, NAME_GameNetDriver))
		{
			NetDriver = GEngine->FindNamedNetDriver(this, NAME_PendingNetDriver);
		}
		// 然后就是初始化链接
		if( NetDriver->InitConnect( this, URL, ConnectionError ) )
		{
			UNetConnection* ServerConn = NetDriver->ServerConnection;

			FNetDelegates::OnPendingNetGameConnectionCreated.Broadcast(this);

			// Kick off the connection handshake
			if (ServerConn->Handler.IsValid())
			{
				ServerConn->Handler->BeginHandshaking(
					FPacketHandlerHandshakeComplete::CreateUObject(this, &UPendingNetGame::SendInitialJoin));
			}
			else
			{
				SendInitialJoin();
			}
		}
		else
		{
			UE_LOG(LogNet, Warning, TEXT("error initializing the network stack"));
			GEngine->DestroyNamedNetDriver(this, NetDriver->NetDriverName);
			NetDriver = NULL;
		}
	}
}
```
不是很复杂，首先就是通过CreateNamedNetDriver创建出NetDriver，这部分和服务器的UWorld::Listen方法一样，服务器之后会调用NetDriver的InitListen方法，而这里客户端会调用NetDriver的InitConnect方法。
### 2 InitConnect
```cpp
bool UIpNetDriver::InitConnect( FNetworkNotify* InNotify, const FURL& ConnectURL, FString& Error )
{
	// 这部分也是初始化，和服务器一样
	// 然后就是设置了FNetworkNotify类型的成员变量Notify，在客户端这部分这个Notify是当前的UPendingNetGame，而服务器是World
	if( !InitBase( true, InNotify, ConnectURL, false, Error ) )
	{
		return false;
	}

	// Create new connection.
	// 注意这个ServerConnection是客户端创建赋值的，表示链接的服务器只有一个
	// 这里创建了ServerConnection这是一个UPlayer的子类
	// 其中NetConnectionClass是在InitBase里面赋值的
	ServerConnection = NewObject<UNetConnection>(GetTransientPackage(), NetConnectionClass);
	UIpConnection* IPConnection = CastChecked<UIpConnection>(ServerConnection);

	// 这里里面初始化一堆数据，PacketHandler，StatelessConnectComponent
	ServerConnection->InitLocalConnection(this, Socket, ConnectURL, USOCK_Pending);

	// 创建Channel
	CreateInitialClientChannels();

	return true;
}
```
这个部分就是通过InitBase初始化一些数据，然后创建出ServerConnection的UPlyaer对象，然后初始化，最后就是创建Channels，Channels是通过BaseEngine.ini配置中ChannelDefinitions这个字段创建的，默认是三个（ControlChannel，VoiceChannel，ActorChannel）
