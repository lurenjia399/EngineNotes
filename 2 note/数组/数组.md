# 1 TStridedView
```cpp
2. MakeStridedView(ZoneStorage.Zones, &FZoneData::Bounds)

  这是关键部分，创建一个"跨步视图"：

  参数1: ZoneStorage.Zones - 所有区域的数组
  参数2: &FZoneData::Bounds - 成员指针，指向每个区域的包围盒

  作用：从数组中提取特定成员，而不复制整个对象

  具体例子

  假设数据结构

  struct FZoneData {
      FString Name;           // 偏移 0
      FBox Bounds;            // 偏移 16  ← 我们要的
      TArray<FLane> Lanes;    // 偏移 40
      // ... 其他数据
  };

  TArray<FZoneData> Zones = {
      { "Zone_A", FBox(...), ... },  // 100字节
      { "Zone_B", FBox(...), ... },  // 100字节
      { "Zone_C", FBox(...), ... },  // 100字节
  };

  MakeStridedView 的作用

  // 传统方式（低效）：需要复制数据
  TArray<FBox> BoundsArray;
  for (const FZoneData& Zone : Zones) {
      BoundsArray.Add(Zone.Bounds);  // 复制！
  }
  BVTree.Build(BoundsArray);

  // StridedView 方式（高效）：零拷贝
  auto View = MakeStridedView(Zones, &FZoneData::Bounds);
  // View 内部：
  // - 起始地址: &Zones[0].Bounds
  // - 步长: sizeof(FZoneData) = 100字节
  // - 数量: 3
  BVTree.Build(View);  // 直接访问原始内存

  内存布局

  内存地址:
  0x1000: [Zone_A.Name | Zone_A.Bounds | Zone_A.Lanes | ...]
                         ↑ 读取这里
  0x1064: [Zone_B.Name | Zone_B.Bounds | Zone_B.Lanes | ...]
                         ↑ 跳过100字节，读取这里
  0x10C8: [Zone_C.Name | Zone_C.Bounds | Zone_C.Lanes | ...]
                         ↑ 再跳过100字节，读取这里

  StridedView = {
      DataPtr: 0x1010,     // 第一个Bounds的地址
      Stride: 100,         // 每次跳过100字节
      Count: 3
  }
```

```cpp
 与 TArray 的区别：

// TArray - 紧凑数组                                    
TArray<FTransform> DenseArray;
DenseArray.Add(Transform1);  // Index 0
DenseArray.Add(Transform2);  // Index 1
DenseArray.RemoveAt(0);      // Transform2 移动到 Index 0

// TSparseArray - 稀疏数组
TSparseArray<FTransform> SparseArray;
int32 Index0 = SparseArray.Add(Transform1);  // Index 0
```

# TConstArrayView
```cpp
// 不拥有数据，只是一个"窗口"
  TConstArrayView<int32> View;  // 只存储：指针 + 长度
  
  与其他容器的关系

  TArray<int32> MyArray = {1, 2, 3, 4, 5};

  // 可以从 TArray 隐式转换
  TConstArrayView<int32> View1 = MyArray;

  // 可以从 C 数组创建
  int32 CArray[] = {1, 2, 3};
  TConstArrayView<int32> View2(CArray, 3);

  // 可以从指针 + 长度创建
  TConstArrayView<int32> View3(MyArray.GetData(), MyArray.Num());

  // 可以从初始化列表创建
  TConstArrayView<int32> View4 = {1, 2, 3};

  使用方式

  void ProcessData(TConstArrayView<FTransform> Transforms)
  {
      // 像数组一样访问
      for (int32 i = 0; i < Transforms.Num(); ++i)
      {
          const FTransform& Transform = Transforms[i];
      }

      // Range-based for
      for (const FTransform& Transform : Transforms)
      {
          // 处理
      }

      // 获取原始指针
      const FTransform* Data = Transforms.GetData();
  }

  // 调用时可以传各种类型
  TArray<FTransform> Array;
  ProcessData(Array);  // OK

  FTransform StaticArray[10];
  ProcessData(StaticArray);  // OK

  ProcessData({Transform1, Transform2});  // OK

  优势

  1. 零拷贝：不复制数据，只传递指针
  2. 类型安全：比 const FTransform*, int32 Num 更安全
  3. 统一接口：可以接受 TArray、C 数组、指针等
  4. 只读保证：调用者知道数据不会被修改
```
